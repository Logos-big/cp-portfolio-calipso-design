# Правила работы с дизайн-системой Figma

## ⚠️ ВАЖНО: Строгое следование макету

**Всегда следуйте ТОЛЬКО строго по макету и дизайн-системе из Figma.**

### Основные правила:

1. **НЕ добавлять ничего, чего нет в макете**
   - ❌ Не добавлять элементы, которых нет в Figma
   - ❌ Не добавлять стили, не указанные в макете
   - ❌ Не добавлять анимации или эффекты, не указанные в дизайне

2. **Точно воспроизводить структуру**
   - ✅ Использовать точно такую же иерархию элементов
   - ✅ Сохранять порядок элементов как в Figma
   - ✅ Использовать те же типы элементов (FRAME, TEXT, RECTANGLE и т.д.)

3. **Точно воспроизводить стили**
   - ✅ Использовать точные размеры из макета
   - ✅ Использовать точные цвета из дизайн-системы
   - ✅ Использовать точные шрифты и их параметры
   - ✅ Использовать точные отступы и промежутки

4. **Следовать дизайн-системе**
   - ✅ Использовать только цвета из палитры Figma
   - ✅ Использовать только шрифты из типографики Figma
   - ✅ Использовать только spacing из дизайн-системы

### Что делает скрипт автоматического обновления:

Скрипт `scripts/auto-update.ps1`:
- ✅ Получает данные ТОЛЬКО из указанного узла Figma (Main)
- ✅ Генерирует HTML строго по структуре макета
- ✅ Применяет стили строго из макета
- ✅ НЕ добавляет ничего лишнего

### Проверка соответствия:

После обновления проверьте:
1. Структура HTML соответствует структуре в Figma
2. Все размеры точно совпадают
3. Все цвета точно совпадают
4. Все шрифты точно совпадают
5. Все отступы точно совпадают

### Если что-то не совпадает:

1. Проверьте, что макет "Main" обновлен в Figma
2. Запустите обновление: `.\scripts\auto-update.ps1`
3. Сравните результат с макетом в Figma

---

## Автоматическое обновление

Когда вы обновляете макет в Figma, просто напишите:
**"обновил Main"**

Скрипт автоматически:
1. Получит свежие данные из Figma
2. Обновит HTML строго по макету
3. Обновит сайт на GitHub Pages
