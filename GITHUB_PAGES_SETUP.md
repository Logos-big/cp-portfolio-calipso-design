# Настройка GitHub Pages (бесплатный хостинг)

## Что такое GitHub Pages?

GitHub Pages — это бесплатный хостинг от GitHub для статических сайтов. Идеально подходит для разработки и тестирования.

## Преимущества:
- ✅ Полностью бесплатно
- ✅ Автоматический деплой при каждом push
- ✅ HTTPS по умолчанию
- ✅ Свой домен (опционально)
- ✅ Быстрая работа

## Шаг 1: Включите GitHub Pages

1. Зайдите на ваш репозиторий: https://github.com/Logos-big/cp-portfolio-calipso-design
2. Нажмите **Settings** (Настройки)
3. В левом меню найдите **Pages** (в разделе "Code and automation")
4. В разделе **Build and deployment** → **Source**:
   - Нажмите на dropdown "Deploy from a branch" (если он есть)
   - Или в dropdown "Branch" выберите `main` (вместо "None")
   - В поле "Folder" выберите `/ (root)` или `/`
5. Нажмите **Save**

**Важно:** Если видите "GitHub Pages is currently disabled" - это нормально, просто выберите branch `main` в dropdown и нажмите Save.

## Шаг 2: Подождите несколько минут

GitHub обработает ваш сайт. Обычно это занимает 1-2 минуты.

## Шаг 3: Откройте ваш сайт

Ваш сайт будет доступен по адресу:
**https://logos-big.github.io/cp-portfolio-calipso-design**

(Или по адресу, который покажет GitHub в настройках Pages)

## Автоматический деплой

Теперь при каждом `git push` ваш сайт автоматически обновится!

```powershell
git add .
git commit -m "Обновление сайта"
git push
```

Через 1-2 минуты изменения появятся на сайте.

## Перенос на платный хостинг

Когда будете готовы перенести на платный хостинг:
1. Заполните `deploy-config.json` данными платного хостинга
2. Выполните `.\scripts\deploy.ps1`
3. Готово! Сайт будет на платном хостинге

Можно даже использовать оба одновременно — GitHub Pages для разработки, платный для продакшена!
