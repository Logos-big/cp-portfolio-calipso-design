# Ошибка 429: Rate limit exceeded в Figma API

## Что это значит?

**Ошибка 429 "Rate limit exceeded"** означает, что вы превысили лимит запросов к Figma API.

Figma API имеет ограничения:
- **Обычно:** 200 запросов в минуту
- **Для бесплатных аккаунтов:** может быть меньше

## Почему это происходит?

1. Слишком частые запросы к API
2. Множественные попытки обновления за короткое время
3. Другие приложения также используют ваш токен

## Как исправить?

### Вариант 1: Подождать (самый простой)

Просто подождите **1-2 минуты** и попробуйте снова:

```powershell
.\scripts\auto-update.ps1
```

### Вариант 2: Использовать существующие данные

Если данные в `figma-data.json` актуальные (недавно обновлены), можно использовать их напрямую без запроса к API.

### Вариант 3: Увеличить интервал между запросами

Если вы часто обновляете, добавьте задержку между запросами.

## Проверка лимитов

Figma API лимиты:
- **Бесплатный план:** ~200 запросов в минуту
- **Профессиональный план:** больше лимитов

## Рекомендации:

1. **Не запускайте скрипт слишком часто**
   - Подождите 1-2 минуты между запросами
   - Обновляйте только когда действительно изменили макет

2. **Используйте кэшированные данные**
   - Если `figma-data.json` недавно обновлен, можно использовать его
   - Проверьте `lastModified` в файле

3. **Проверьте другие приложения**
   - Убедитесь, что другие скрипты/приложения не используют тот же токен одновременно

## Быстрое решение:

**Просто подождите 1-2 минуты и запустите снова:**

```powershell
.\scripts\auto-update.ps1
```

Или используйте существующие данные из `figma-data.json` - они могут быть актуальными.
