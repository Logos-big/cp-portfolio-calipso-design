# Инструкция по подключению хостинга

## Шаг 1: Получите данные FTP от хостинга

Обычно хостинг-провайдер отправляет данные в письме или они доступны в панели управления. Вам понадобятся:

- **FTP хост** (например: `ftp.ваш-сайт.com` или IP адрес)
- **FTP порт** (обычно `21` для FTP или `22` для SFTP)
- **Логин** (FTP пользователь)
- **Пароль** (FTP пароль)
- **Путь к папке сайта** (обычно `/public_html` или `/www` или `/htdocs`)

## Шаг 2: Создайте файл конфигурации

1. Скопируйте файл `deploy-config.json.example` в `deploy-config.json`:
   ```powershell
   Copy-Item deploy-config.json.example deploy-config.json
   ```

2. Откройте `deploy-config.json` и заполните ваши данные:

```json
{
  "host": "ftp.ваш-хостинг.com",
  "port": 21,
  "user": "ваш_логин",
  "password": "ваш_пароль",
  "remotePath": "/public_html",
  "localPath": "./",
  "exclude": [
    ".git",
    ".gitignore",
    "node_modules",
    "deploy-config.json",
    "*.md",
    ".DS_Store"
  ]
}
```

**Важно:** Файл `deploy-config.json` уже добавлен в `.gitignore`, чтобы пароли не попали в GitHub!

## Шаг 3: Выполните деплой

Запустите скрипт деплоя:

```powershell
.\scripts\deploy.ps1
```

Или с указанием пути к конфигу:

```powershell
.\scripts\deploy.ps1 -ConfigPath "deploy-config.json"
```

## Шаг 4: Проверьте сайт

После успешного деплоя откройте ваш сайт в браузере и проверьте, что все работает.

## Альтернативные способы деплоя

### Через FileZilla (графический интерфейс)

1. Скачайте [FileZilla](https://filezilla-project.org/)
2. Откройте FileZilla
3. Введите данные FTP:
   - Хост: `ftp.ваш-хостинг.com`
   - Логин: `ваш_логин`
   - Пароль: `ваш_пароль`
   - Порт: `21`
4. Подключитесь
5. Перетащите файлы из локальной папки в папку `public_html` на сервере

### Через SSH (если доступен)

Если у вас есть SSH доступ, можно использовать:

```powershell
scp -r * user@host:/path/to/public_html
```

## Решение проблем

### Ошибка подключения
- Проверьте правильность данных FTP
- Убедитесь, что порт правильный (21 для FTP, 22 для SFTP)
- Проверьте, не блокирует ли файрвол подключение

### Файлы не загружаются
- Проверьте права доступа на папку на сервере (должны быть 755 для папок, 644 для файлов)
- Убедитесь, что путь `remotePath` правильный

### Сайт не открывается
- Проверьте, что `index.html` находится в корневой папке сайта
- Убедитесь, что путь к CSS правильный (`styles/main.css`)
